**Presenter:** Adrian Tennant

> "Traditional storytelling tools give you a blank page. Even with AI today, you're still prompting for text, prompting for images separately, wiring up audio manually, building the interactive layer yourself. Hours — if not days."

> "Narrative Alchemist is different. Not a single AI prompt. A **coordinated multi-agent pipeline** — each agent owns one layer of the creative stack, and they hand off to each other automatically."

## WALKTHROUGH — AGENT PIPELINE (75 seconds)

*(Dashboard loading — agents running live)*

> "Watch the pipeline. Four agents, two stages, all coordinated."

> "**PlotAgent** runs first. It doesn't hallucinate the story — it sends a query to our **Azure AI Search** index, retrieves verified real-world facts about the setting, then uses those facts to architect the story outline: title, theme, scene stubs. Grounded from the very first step."

*(PlotAgent → Done)*

> "**WorldAgent** now takes each scene stub and expands it in parallel — three scenes, three concurrent calls to **Azure OpenAI GPT-4o**. For every scene it writes: immersive second-person narrative prose, a detailed image generation prompt, and a narration script. Three outputs per scene, all in one structured response."

*(WorldAgent → Done)*

> "**MediaAgent** takes each image prompt and fires it at **GPT Image 2** — generating a full 1024×1024 illustration per scene. These run in parallel too. Watch them appear in the asset panel on the right."

*(Images populating)*

> "And **CodeAgent** acts as the assembler — it validates all assets, bundles everything into a typed `StoryBundle` object, and writes it to disk so the viewer can load it instantly."

*(Progress bar hits 100% — Open Story Viewer button appears)*

> "45 seconds. Three scenes, three illustrations, narration scripts, branching choices — done."

## SYSTEM ARCHITECTURE (40 seconds)

> "Let me walk through the full stack quickly."

> "The frontend is **Next.js 15** with the App Router, TypeScript, and Tailwind CSS. The agent pipeline runs entirely in Next.js API routes — no separate backend service."

> "The four agents each map to a dedicated API route: `/api/generate` orchestrates the chain via SSE, `/api/tts` handles neural voice. Each agent is a typed async function that calls Azure services and emits structured JSON."

> "Azure services used:"

> "— **Azure OpenAI GPT-4o** powers PlotAgent and WorldAgent for story intelligence  
> — **GPT Image 2** powers MediaAgent for scene illustrations  
> — **Azure AI Search** with a Foundry IQ knowledge index for real-world fact grounding  
> — **Azure AI Speech** Aria Neural for scene narration"

> "Agent handoff is explicit and typed — PlotAgent returns a `StoryOutline`, WorldAgent returns `SceneAssets[]`, MediaAgent returns `ImageResult[]`, CodeAgent assembles the final `StoryBundle`. No loose JSON passing between prompts."

## Backup Q&A Answers

**Q: How long does generation take?**  
A: About 45–90 seconds for a 3-scene story. Most of that is GPT Image 2 — 3 parallel image calls. Text generation is under 15 seconds.

**Q: Can it do longer stories?**  
A: Yes — Medium (5 scenes) and Long (8 scenes) are supported. Long stories take about 2–3 minutes.

**Q: What's in the Azure AI Search index?**  
A: Indexed with Caribbean climate and regional knowledge data. The same pattern works with any domain — swap the index, the grounding changes accordingly.

**Q: Is narration real Azure Speech or browser TTS?**  
A: Real Azure AI Speech — Aria Neural. `/api/tts` calls the Azure Speech endpoint server-side and streams the MP3 back to the browser.

**Q: What happens if image generation fails?**  
A: Pipeline continues gracefully — scenes show a styled placeholder. Text, audio, and choices all still work.

**Q: Why Server-Sent Events instead of WebSockets?**  
A: SSE is unidirectional and works natively with Next.js API routes — no separate socket server needed. Pipeline progress is one-directional (server → browser), so SSE is the right fit.
